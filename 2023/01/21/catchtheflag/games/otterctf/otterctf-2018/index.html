内存取证

#内存取证

What the password?

  • 题目描述

you got a sample of rick’s PC’s memory. can you get his user password?

  • 题目解析

题目要我们去获取用户的密码

先获取下内存的系统版本

python3 vol.py -f OtterCTF.vmem windows.info

目标为用户密码，这里使用 lsadump 读取用户密码

python3 vol.py -f OtterCTF.vmem windows.lsadump

获得用户密码

MortyIsReallyAnOtter

General Info

#注册表 #主机名 #IP地址

  • 题目描述

Let’s start easy - whats the PC’s name and IP address?

  • 题目解析

让我们去找 PC 的名字以及 IP 地址

在注册表 MACHINE\SYSTEM\ControlSet001\Control\ComputerName\ComputerName 字段中存有主机名

python3 vol.py -f OtterCTF.vmem windows.registry.printkey  --key "ControlSet001\Control\ComputerName\ComputerName"

获得主机名 WIN-LO6FAF3DTFE

获取 IP 地址可以扫描下网络信息

python3 vol.py -f OtterCTF.vmem windows.netscan

获得 IP 地址 192.168.202.131

组合 flag 为 WIN-LO6FAF3DTFE-192.168.202.131

Play Time

#进程列表

  • 题目描述

Rick just loves to play some good old videogames.
can you tell which game is he playing?
whats the IP address of the server?

  • 题目解析

要去找到游戏名以及游戏服务器的 IP 地址

查看下进程列表

python3 vol.py -f OtterCTF.vmem windows.pslist

找到游戏名为 LunarMS

根据 PID 查看网络连接

python3 vol.py -f OtterCTF.vmem windows.netscan | findstr "708"

获得游戏服务器 IP 地址为 77.102.199.102

组合获得 flag 为 LunarMS-77.102.199.102

  • 题目描述

Silly rick always forgets his email’s password, so he uses a Stored Password Services online to store his password. He always copy and paste the password so he will not get it wrong. whats rick’s email password?

  • 题目解析

根据题目可知将密码复制到了剪切板中

这里使用 volatility2 中的 clipboard 模块读取剪切板

获得 flag 为 M@il_Pr0vid0rs

Name Game

  • 题目描述

We know that the account was logged in to a channel called Lunar-3. what is the account name?

  • 题目解析

在登录过程中，账号可能会以明文的形式储存在内存之中，我们便可以利用 grep 指令筛选出所有的可打印句子，然后通过关键词 Lunar-3 来定位账号名可能会出现的地方

将 LunarMS.exe 进程的内存数据导出来

volatility_2.6_win64_standalone.exe -f OtterCTF.vmem --profile Win7SP1x64 memdump --pid 708 --dump-dir ./output/

读取其中的字符串

strings 708.dmp  | grep -C 10 "Lunar-3"

获得 flag 为 0tt3r8r33z3

Hide And Seek

  • 题目描述

The reason that we took rick’s PC memory dump is because there was a malware infection. Please find the malware process name (including the extension)

  • 题目解析

翻译一下就是找出恶意进程的名字

进程就那几个，一个一个尝试都行

可以看到，看似无害的 vmware-tray.exe 进程，他的父进程是 Rick And Morty.exe，这是非常不对劲的，可以怀疑这个 vmware-tray.exe 进程实际上是恶意软件释放出来的子进程

通过cmdline模块，可以追踪到程序的调用指令

这里得到的 flag 为 vmware-tray.exe

Name Game 2

  • 题目描述

From a little research we found that the username of the logged on character is always after this signature: 0x64 0x??{6-8} 0x40 0x06 0x??{18} 0x5a 0x0c 0x00{2}
What’s rick’s character’s name?

  • 题目解析

提取之前导出的 LunarMS.exe 进程的内存数据的十六进制值并进行匹配

hexdump -C 708.dmp | grep -C 1 "5a 0c 00 00"

获得 flag 为 M0rtyL0L

Path To Glory

  • 题目描述

How did the malware got to rick’s PC? It must be one of rick old illegal habits…

  • 题目解析

之前在进程列表中发现存在种子下载工具，推测为通过种子下载下来恶意文件

全局搜索一下种子位置

python3 vol.py -f OtterCTF.vmem windows.filescan | grep "Rick And Morty"

导出来查看其文件内容

python3 vol.py -f OtterCTF.vmem windows.dumpfile --physaddr 0x7dae9350

读取其内容

strings "file.0x7dae9350.0xfa801b42c9e0.DataSectionObject.Rick And Morty season 1 download.exe.1.torrent.dat"

获得 flag 为 M3an_T0rren7_4_R!ck

Recovery

  • 题目描述

Rick got to have his files recovered! What is the random password used to encrypt the files?

  • 题目解析

要我们获得用于加密的密码

分析exe可以看到两个函数，第一个创建密码是随机的，然后有一个发送密码，是把computerName和userName还有password拼接以后进行发送

可以分析出compuerName就是WIN-LO6FAF3DTFE，用户名是Rick，所以这个函数发送的字符串应该是 WIN-LO6FAF3DTFE-Rick password，我们直接去这个勒索软件的内存中寻找相关字符串即可

strings -el ./3720.dmp | grep 'WIN-LO6FAF3DTFE-Rick' -C 5

获得 flag 为 aDOBofVYUNVnmp7

  • 题目描述

There’s something fishy in the malware’s graphics.

  • 题目解析

使用 foremost 分离应用获得图片

foremost executable.3720.exe

获得 flag 为 S0_Just_M0v3_Socy

Closure

  • 题目描述

Now that you extracted the password from the memory, could you decrypt rick’s files?

  • 题目解析

被加密的文件为 flag.txt ，先把 flag.txt 导出

volatility_2.6_win64_standalone.exe  -f OtterCTF.vmem --profile Win7SP1x64 dumpfiles -Q 0x000000007e410890 -D ./output

将末尾多余的 0 都去掉然后修改后缀为 locked

使用解密工具 HiddenTearDecrypter 使用之前获得的密码进行解密，建议再虚拟机里面使用


获得 flag 为 {Im_Th@_B3S7_RicK_0f_Th3m_4ll}
