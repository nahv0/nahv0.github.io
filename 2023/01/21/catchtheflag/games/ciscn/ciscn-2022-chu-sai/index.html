5. online_crt

#OpenSSL #SSRF #c_rehash #CVE-2022-1292

  • 题目描述

无

  • 题目解析

题目给出了附件

审计了一下主要是一个证书的操作

在 /getcrt 中根据用户信息生成对应的证书

在 /createlink 中使用 c_rehash 进行处理证书

c_rehash 为文件创建一个符号连接，并将此符号连接的名称设为文件的 hash 值，作用是让 openssl 在证书目录中能够找到证书。

并且题目将改文件直接给了出来，经过搜索发现该文件存在命令注入漏洞

OpenSSL c_rehash 命令注入漏洞

找到了漏洞利用点，但是由于证书的名称是随机生成的，所以还需要找个方法修改文件名

在 /proxy 中可以请求另一个服务，同时在 8887 端口还开放了一个 web 服务，审计一下发现可以在这个服务中进行修改文件名，可以利用 CRLF 构造任意请求

还需要绕过一个 if 判断

if c.Request.URL.RawPath != "" && c.Request.Host == "admin"

这里的 RawPath 查看源代码可知保存的是在进行请求时目标地址被编码过的内容，没有被编码则该值为空，所以这里还需要对请求的内容 URL 编码一下

由于没有回显所以需要外带构造 payload

uri=/admin%252frename?oldname=97dbf242-253c-4a27-a134-a060eb894649.crt%26newname=`echo%2520Y2F0IC8qPjE%253d|base64%2520-d|bash`.crt%20HTTP/1.1%0d%0aHost:%20admin%0d%0a%0d%0aGET%20/

这里读取到的为 form 中的内容，所以还需要加一个

Content-Type: application/x-www-form-urlencoded

然后访问 /createlink 触发命令注入然后访问 /static/crt/1 下载下来结果
